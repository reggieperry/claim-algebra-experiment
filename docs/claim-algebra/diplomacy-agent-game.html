Diplomacy as a claim-algebra agent game

Design and actor-model implementation — seven LLM powers negotiating and moving over a verifiable substrate, with the inter-agent message as a typed claim.
Companion to claim-algebra.html (§9, the adversarial instance) · the implementation spec for hand-off is diplomacy-implementation.md · math via MathJax, diagrams via Mermaid

The claim algebra has two cooperative instances — the credit workbench (a confidence semiring) and an agent fulfillment network (a tropical cost semiring). This document develops the third, adversarial instance in full: a game of Diplomacy played by seven LLM agents, where the board and its adjudication are the claim algebra working at its cleanest, and the negotiation is the claim algebra meeting its honest boundary. It then lands the design on the actor model as the communication layer and describes the game loop concretely enough to build from.

1 · The game and its rules

Diplomacy (Calhamer, 1959 [1]) is a seven-player game on a map of 1901 Europe. Its defining property — and the reason it is the standard AI-negotiation benchmark — is that it has no dice, no cards, and no hidden draws: every outcome is decided entirely by the orders and how they interact. All variance is in the negotiation and in predicting the others.

  • Powers (7): England, France, Germany, Italy, Austria-Hungary, Russia, Turkey.
  • Map: ~75 provinces (land, sea, coastal). 34 are Supply Centers (SCs). Each power begins on its home SCs with 3 units (Russia 4). Units are Armies (land) or Fleets (sea / coast).
  • Goal: control 18 of the 34 SCs — a majority — for a solo win; otherwise a negotiated draw among survivors.
  • Simultaneity: all powers write orders secretly, then all orders are revealed and adjudicated at once.

The four order types. Every unit issues exactly one:

OrderMeaning and rule
HoldStay in place (defensive strength 1, plus any support).
Move (attack)Attempt to move to an adjacent province. An army moves to adjacent land; a fleet to an adjacent sea or along coasts.
SupportAdd +1 strength to another unit's hold or move. The supporter must itself be able to reach the target province. Support is cut if the supporter is attacked from any province other than the one it is supporting into.
ConvoyA fleet (or a chain of fleets) in sea provinces carries an army across the water from one coast to another.

Resolution. Base strength is 1; each valid support adds 1. A move succeeds only if it out-strengths the opposition; equal strength is a standoff (a bounce — nobody moves). A unit cannot dislodge one supported to equal strength, and cannot dislodge its own unit. A dislodged unit must retreat to an empty adjacent province (not the one it was attacked from, not a province that stood off) or be disbanded.

The year. Spring then Fall, each a movement phase followed by a retreat phase. After Fall comes an adjustment phase: count SCs — control more SCs than you have units and you build on a vacant home SC; control fewer and you disband.

Negotiation. Before orders, powers negotiate freely — alliances, non-aggression pacts, coordinated attacks. Crucially, negotiation is non-binding: there is no enforcement, and betrayal is a feature of the game, not a violation of it. The full rules' worth of adjudication intricacy (support-cutting, beleaguered garrisons, convoy paradoxes) is exactly specified by the community's DATC adjudicator test cases [2], which any engine must pass.

2 · The communication substrate and the mathematics

This is the third instance of the claim algebra, and the only one that straddles the boundary the parent document draws. It separates into two regimes that meet at adjudication.

2.1 · Two regimes, three claim types

Everything an agent emits is a Claim[K, A] from the parent document. Diplomacy uses two semirings \(K\) and three claim types:

Claim typeSemiring \(K\)What it isVerification
StateClaimVerification (Boolean)A board fact — who holds which province, where each unit is.Immediate, against the engine. Fail-closed: a board fact that does not verify is rejected; no power can hallucinate the board.
OrderVerification (Boolean)A Hold / Move / Support / Convoy commitment, submitted privately.Legality is checked; an illegal order hits the annihilator and is dropped to a Hold. Adjudication is deterministic.
DealTrust (graded bilattice)A negotiation message — a conditional promise ("I support you into Galicia if you move there").Deferred — resolved only at adjudication ("did your orders match your pledge?"). Feeds reputation.

2.2 · The substrate is the claim algebra proper

Board, orders, and adjudication are a clean verification-semiring instance. Adjudication is a pure, deterministic function

$$ \mathrm{adjudicate} : \mathrm{Orders} \times \mathrm{Board} \longrightarrow (\mathrm{Board}', \mathrm{Outcomes}), $$

and the engine is the verification oracle. Every resulting board fact is a verified derivation from the submitted orders — so the whole substrate is fail-closed and auditable. The annihilator does real work here: an illegal or unverifiable order contributes nothing (it becomes a Hold), and a power that cannot confirm a fact cannot assert it.

2.3 · Negotiation — the trust model is a graded bilattice value

A Deal is a promise — a claim whose verification is deferred. At assertion time it is UNRESOLVED-as-fact; the algebra refuses to let cheap talk pose as a board fact. What a receiving power attaches is a trust grade, and that grade is not a number bolted on from outside the algebra: subjective logic embeds in the graded bilattice as its \([0,1]^2\) instance — belief the for-channel, disbelief the against-channel, uncertainty measuring gap-ness — so trust is the algebra's own graded form. (Calibration stays exogenous: the algebra propagates the leaf numbers, it does not produce them.) The opinion is the familiar subjective-logic triple [3]:

$$ \omega = (b,\, d,\, u,\, a), \qquad b + d + u = 1, \qquad \mathbb{E}[\omega] = b + a\,u, $$

belief / disbelief / uncertainty over "will this power keep its word", with base rate \(a\) and expected trust \(b + au\); the opinion occupies the consistent simplex, and a betrayal — a pledge corroborated with its contradicting adjudication — climbs to the glut \(\mathbf{B}\) that a single opinion cannot carry. The opinion is updated by reputation: each adjudicated round pours evidence into the for- or against-channel, moving a power's standing in the knowledge order — toward reliably-keeps (\(\mathbf{T}\)), reliably-defects (\(\mathbf{F}\)), or says-one-thing-does-another (\(\mathbf{B}\)). Reputation is a value in the algebra, not an annotation beside it.

Where the boundary sits. The representation of trust is inside the algebra — a power's reliability, the distinction between "no read yet" (gap) and "caught them lying" (glut), and the evidence accumulated across rounds all live as graded Belnap values. What stays outside is the selection of action — whether to keep your own promise, whom to deceive, when to defect — which is game theory and opponent modelling. A power cannot lie about the board or move illegally (those are verified, fail-closed), but it can lie in negotiation, which is cheap talk and is the game. The algebra shrinks the attack surface to exactly the strategic layer where deception belongs, holds and propagates the trust state, and leaves a replayable provenance DAG; it cannot tell you whether to be trustworthy.

3 · How a turn proceeds

A phase is a deferred-verification loop. Negotiation produces trust-claims; orders produce verification-claims; adjudication is the moment every promise becomes checkable; reputation closes the loop.

Sub-phases (retreat after a movement phase, build after Fall) follow the same submit-then-adjudicate pattern. After each adjudication the GameMaster checks for a solo (18 SCs), eliminations, or an agreed draw.

4 · The Scala realization

The types specialize the parent document's Claim[K, A] to the two semirings. The substrate is pure and deterministic; the agent is the only place an LLM is called.

// ── The verifiable substrate (claim algebra proper) ──────────────────────────
enum Power { case England, France, Germany, Italy, Austria, Russia, Turkey }
enum Kind  { case Army, Fleet }
enum Season { case Spring, Fall }
final case class UnitAt(power: Power, kind: Kind, prov: ProvId)

enum Order:                                            // the four order types
  case Hold(at: ProvId)
  case Move(from: ProvId, to: ProvId, viaConvoy: Boolean = false)
  case Support(at: ProvId, target: Order)              // support a Hold or a Move
  case Convoy(fleet: ProvId, army: Order.Move)

final case class BoardState(
  units:  Set[UnitAt],
  owners: Map[ProvId, Power],                          // SC ownership
  year:   Int, season: Season)

final case class Outcome(order: Order, result: Result) // Succeeded | Bounced | Cut | Dislodged | Illegal
final case class Outcomes(perOrder: Map[(Power, Order), Outcome])

// Adjudication is the verification ORACLE: pure, deterministic, DATC-correct.
def adjudicate(orders: Map[Power, Set[Order]], s: BoardState): (BoardState, Outcomes)

type StateClaim = Claim[Verified, UnitAt]              // verify() checks against the engine

// ── The diplomacy layer (action selection — the strategic layer outside the algebra) ───────
final case class Opinion(b: Double, d: Double, u: Double, base: Double):
  def expected: Double = b + base * u                  // trust value in [0,1]
  def fuse(ev: Opinion): Opinion                       // subjective-logic cumulative fusion (corroborate)
  def discount(via: Opinion): Opinion                  // trust through a third party

final case class Reputation(of: Map[Power, Opinion]):
  // promise-vs-action: every Deal the other made, scored against what they actually ordered
  def observe(power: Power, promised: Set[Order], actual: Set[Order]): Reputation

final case class Deal(from: Power, to: Power, iPledge: Set[Order], iExpect: Set[Order])
type DealClaim = Claim[Opinion, Deal]                  // K = trust; UNRESOLVED-as-fact until adjudication

// ── The agent: the only LLM seam (bounded, structured calls) ─────────────────
trait PowerAgent:
  def me: Power
  def negotiate(view: BoardState, rep: Reputation, inbox: Vector[Deal]): Vector[Deal]   // LLM
  def chooseOrders(view: BoardState, rep: Reputation, deals: Vector[Deal]): Set[Order]  // LLM
  def reflect(rep: Reputation, promised: Map[Power, Set[Order]], outcomes: Outcomes): Reputation

Three things are worth noting. The adjudicate function is the spine and must pass the DATC suite [2] — it is the verification oracle the whole game trusts. chooseOrders returns legal-by-construction-or-dropped orders; the adjudicator re-checks. And reflect turns the gap between iPledge and the actual orders into reputation — the deferred verification of every Deal, made into an opinion update.

5 · The actor model as the communication layer

The actor model is the natural substrate: each power has private state (its secret intentions and its model of everyone else), the press is message-passing, adjudication is a serialization point, and the whole game is a durable, replayable log. This is the same substrate the focused-harness actor-model design develops; here it is specialized to a turn-based game with a referee.

5.1 · The actors

ActorResponsibilityState
GameMasterThe phase state machine and clock — opens negotiation windows, sets order deadlines, sequences movement → retreat → build, checks victory/draw. Orchestrates; does not adjudicate.Phase, deadlines, the canonical board (as verified StateClaims).
AdjudicatorThe pure rules engine and verification oracle. Receives the full order set, resolves it atomically, emits the new board + outcomes. DATC-correct.None — a pure function behind an actor for serialization.
PowerAgent ×7An LLM-backed player. Negotiates, chooses orders, updates its reputation model. Isolated: its secret intentions never leak except via messages it chooses to send.Private: opponent model, reputation opinions, draft orders.
PressRouterRoutes Deal / message traffic between powers — private channels, and optional public ("white"), anonymous ("grey"), or forged-attribution ("black") press.Channel topology; message log.
ChronicleAppend-only durable log of every claim and adjudication — the provenance DAG. Enables replay, audit, crash recovery, and evals.The full ordered event history.

5.2 · The actor-model properties that earn their keep

  • Isolation = private information. Diplomacy is a game of secrets. Each PowerAgent's intentions and opponent model are its own state, unreachable except through messages it elects to send. The actor boundary is the information boundary.
  • Fail-closed via NMR. A PowerAgent that misses the order deadline (crash, timeout, or a model that stalls) defaults to all Holds — the standard "No Moves Received" rule. A silent agent can neither stall the game nor be exploited beyond holding. This is the annihilator at the orchestration level.
  • Supervision. A crashed PowerAgent is restarted from the Chronicle's public state and resumes next phase; the game never blocks on one player.
  • The serialization window. Orders arrive asynchronously and accumulate; the Adjudicator then performs one atomic, deterministic resolve. This is the lease-scoped serialization the actor-model design names — the Adjudicator holds the resolve lease, and nothing observes a half-resolved board.
  • Durability and replay. The Chronicle is append-only and the Adjudicator is pure, so the entire game replays deterministically — the provenance DAG that makes the game auditable, debuggable, and usable as an eval/training corpus.
  • Location transparency. PowerAgents may run on different LLM providers or hosts; the GameMaster does not care, which makes mixed-model tournaments trivial.

6 · The game loop, in code

The loop is an actor choreography driven by the GameMaster's phase state machine. In a typed-actor style (Akka/Pekko Typed), the protocol is an ADT of messages and the GameMaster is a behavior that advances on timers and on receipt of all orders.

// ── The actor message protocol (the wire = Claim values + control) ───────────
enum GMsg:
  case Register(power: Power, ref: ActorRef[AgentMsg])
  case OrdersFrom(power: Power, orders: Set[Order])      // a PowerAgent's submission
  case Resolved(board: BoardState, outcomes: Outcomes)   // from the Adjudicator
  case Tick                                              // the clock (deadlines)

enum AgentMsg:
  case PhaseStart(board: BoardState, phase: Phase, negotiateBy: Deadline, orderBy: Deadline)
  case Inbound(deal: Deal)                               // a Deal routed by the PressRouter
  case Adjudicated(board: BoardState, outcomes: Outcomes)

enum AdjMsg:
  case Resolve(orders: Map[Power, Set[Order]], board: BoardState, replyTo: ActorRef[GMsg])

// ── The GameMaster: the phase state machine ──────────────────────────────────
def gameMaster(s: GameState): Behavior[GMsg] = Behaviors.receive { (ctx, msg) => msg match
  case Tick if s.phase == Negotiating && past(s.negotiateBy) =>
    s.powers.foreach(_ ! PhaseStart(s.board, s.phase, expired, s.orderBy))   // open the order window
    gameMaster(s.copy(phase = Ordering))

  case OrdersFrom(p, os) =>
    val s2 = s.recordOrders(p, legalize(os, s.board))                        // illegal → Hold (annihilator)
    if s2.haveAllOrders || past(s.orderBy) then
      val full = s2.ordersOrHold(s.powers)                                   // NMR: missing power → all Holds
      ctx.spawnAdjudication(full, s2.board)                                  // Resolve → Adjudicator
      gameMaster(s2.copy(phase = Adjudicating))
    else gameMaster(s2)

  case Resolved(board2, outcomes) =>
    chronicle.append(board2, outcomes)                                       // durable provenance
    s.powers.foreach(_ ! Adjudicated(board2, outcomes))                      // broadcast verified state
    s.checkEnd(board2) match                                                 // 18 SCs solo? draw? eliminations?
      case Some(result) => finish(result)
      case None         => gameMaster(s.advancePhase(board2))                // retreat / build / next season
}

// ── A PowerAgent: bounded LLM calls behind the actor ─────────────────────────
def powerAgent(me: Power, st: AgentState, gm: ActorRef[GMsg]): Behavior[AgentMsg] =
  Behaviors.receive { (ctx, msg) => msg match
    case PhaseStart(board, phase, negBy, ordBy) =>
      val deals = st.agent.negotiate(board, st.rep, st.inbox)                // LLM seam #1
      deals.foreach(d => press ! Route(d))                                   // emit diplomacy
      ctx.scheduleOnce(until(ordBy)) { gm ! OrdersFrom(me, st.agent.chooseOrders(board, st.rep, st.inbox)) } // LLM seam #2
      powerAgent(me, st.clearInbox, gm)
    case Inbound(deal) => powerAgent(me, st.addInbox(deal), gm)
    case Adjudicated(board, outcomes) =>
      val rep2 = st.agent.reflect(st.rep, st.lastPromisesSeen, outcomes)     // deferred verification → reputation
      powerAgent(me, st.copy(rep = rep2), gm)
  }
Where the discipline lives. legalize is the per-order annihilator (illegal → Hold); ordersOrHold is the NMR fail-closed (a missing power holds); the Adjudicator is the serialization window and verification oracle; chronicle.append is the provenance DAG; and reflect is the deferred verification of every promise turned into a reputation update. The LLM is called in exactly two bounded places — negotiate and chooseOrders — and it can lie in the first but not falsify the board or the rules in the second.

A first milestone needs none of the LLM: build the Adjudicator and pass a DATC subset, then drive it with scripted agents, then swap in LLM negotiate/chooseOrders. The hand-off build spec — milestones, data model, the Adjudicator contract, the LLM-call contracts, fail-closed rules, and acceptance tests — is diplomacy-implementation.md.
