A claim calculus

Revised operational meta-theory of the claim algebra
Formal revision · 22 July 2026
Abstract. The claim algebra supplies candidate-preserving testimonies, provenance operations, a rendered bilattice, and a signing gate. This calculus supplies the dynamic layer: a deterministic transition system over ordered evidence events. It proves determinacy, finite-fold termination, assertion-permutation invariance, withdrawal idempotence, non-fabrication, exclusivity, fail-closed propagation, and relative soundness.
Meta-theoretic correction. The original evaluator was a deterministic left fold, so “confluence” was not the right name for invariance under reordered inputs. This revision defines consuming configurations, uses finite-fold termination, and calls the commutative-fragment result permutation invariance.

1 · Preliminaries

Let \(X\) be uniquely issued lineage tokens and \(\mathbb N[X]\) the free commutative provenance semiring. A normalized testimony is \(t:A\rightharpoonup_f\mathbb N[X]^2\).

The structural corner is slot-level: \(\mathbf B\) means total pro and con are both nonzero, possibly on different candidate keys. Candidate-local conflict is the stronger condition that one key has both channels nonzero.

Corroboration \(\boxplus\) adds both channels pointwise. Refutation swaps channels. Strike clears pro and folds all prior pro and con into con:

$$\operatorname{strike}(p,c)=(0,p+c).$$

Supersession is:

$$\operatorname{supersede}(t,g)=\operatorname{Repl}(\operatorname{strike}t,g).$$

It does not use involutive negation.

2 · Syntax and authorization

Event[A] ::= Assert(g) | Supersede(g) | Withdraw | WithdrawToken(l)
Belief[A] ::= Open(t) | Repl(struck, operative)

The initial belief is \(b_0=\operatorname{Open}(\varepsilon)\). A term is a finite ordered event sequence.

Definition 2.1 · Well-formed and authorized event

Write \(\Gamma\vdash e\;\mathsf{event}\) when every lineage token is authentic, uniquely issued, source-bound, and admissible; every derived polynomial uses declared operations; the actor is authorized for the slot; and supersession or withdrawal is authorized to retire the named state or token. The constant \(1\) is permitted only for a declared nullary axiom. Malformed events are rejected before reduction.

4 · Operational meta-theory

Theorem 4.1 · Determinacy proved

For every well-formed belief and event, exactly one update is defined. The belief/event cases are finite, disjoint, and exhaustive; supersession side conditions are decidable and ordered.

Theorem 4.2 · Finite-fold termination proved

Every finite term reaches a unique final configuration in exactly its length many transitions. Each transition consumes one event, so the remaining-length measure decreases by one.

5 · Permutation invariance and order sensitivity

Theorem 5.1 · Assertion permutation invariance proved

An assertion-only sequence depends only on the multiset of asserted testimonies, because evaluation is a fold of the commutative monoid \((\boxplus,\varepsilon)\).

Non-theorem 5.2 · Full-language permutation invariance false by design

Supersession and whole-slot withdrawal are chronological. In general, asserting then superseding differs from superseding into an empty slot then asserting. Assertions may merge commutatively, but supersessions and withdrawals must be serialized per slot.

6 · Safety

Theorem 6.1 · Whole-slot withdrawal idempotence proved

$$\operatorname{Withdraw}(\operatorname{Withdraw}(b))=\operatorname{Withdraw}(b).$$

This is idempotence, not unrestricted absorption. A lawful later supersession may install a fresh operative testimony.

Remark 6.2 · Rehabilitation asymmetry

After whole-slot withdrawal, reasserting the same value adds pro beside persistent con and blocks as a glut. Supersession can install a fresh operative value. Token withdrawal leaves no con residue, so a later fresh assertion may sign cleanly.

Theorem 6.3 · Non-fabrication proved

If a term signs value \(v\), some admitted assertion or supersession supplied live pro support for \(v\). No transition creates a new pro candidate.

Theorem 6.4 · Exclusivity proved

No signed testimony contains con support anywhere in the slot. Signing requires structural \(\mathbf T\); zero-sum-freeness then forces every con summand to zero.

Theorem 6.5 · Fail-closed propagation proved

If any required leaf in a derivation tree evaluates to the empty testimony, every ancestor required derivation is empty because no candidate pair can be formed. The root gate blocks.

7 · Soundness

Theorem 7.1 · Relative soundness conditional

If every live contributor to the signed value is faithful to an external ground-truth assignment, the signed value is correct. A signed belief has exactly one pro-supported candidate and no con evidence; distinct live contributor values would cause ambiguity and block.

Corollary 7.2 · Conflict blocking and error localization

If a term signs an incorrect value, at least one live contributor supplied that incorrect value. Rival live candidates and represented con evidence block.

No statistical non-amplification theorem. Correlated wrong events may agree; duplicated lineage may inflate a non-idempotent grade; and one bad upstream value may fan out. Error-rate claims require empirical or probabilistic assumptions.

Non-theorem 7.3 · Absolute soundness false

A wrong but internally self-consistent source may pass its verifier and gate. The frequency of such events is the confidently-wrong-at-signature rate.

8 · Adequacy and implementation conformance

Proposition 8.1 · Algebra–calculus adequacy by construction

The transition system computes the same ordered algebraic fold and reads the same gate as the algebra specification.

Obligation 8.2 · Executable implementation conformance separate obligation

A codebase conforms only if event admission, authorization, normalization, corroboration, strike, supersession, token withdrawal, serialization, rendering, verification, and gating refine these definitions. Tests are evidence, not by themselves a proof of equivalence.

Appendix A · Commitment kind

A commitment is \(c=(a,b,\pi,h,\gamma?)\), with committer, counterparty, committer-owned order pattern, horizon, and optional counterparty guard.

$$U(c)=\langle\pi\in\operatorname{resolved}(a,h)\rangle.$$

\(U\) is a forgetful projection, not an embedding: it discards counterparty, guard, consent, and lifecycle state, and need not be injective.

The extended grammar contains propose, accept, rescind, supersede, and resolve. Accepted commitments do not admit ordinary whole-slot withdrawal. Determinacy, lifecycle termination, restriction-and-extension preservation, and reputation-order agreement require direct proofs for the extended system; they do not transfer merely through \(U\).
